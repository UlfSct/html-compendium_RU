• <!DOCTYPE html>
Предназначен для указания типа текущего документа, для меня всегда html.


• <!-- -->
Предназначен для добавления комментариев внутри html-разметки.


• <a>
Предназначен для создания ссылок.

accesskey - активация ссылки с помощью комбинации клавиш.
<a accesskey="c">...</a>
В разных браузерах работает по-разному:
Chrome: Alt + c
Opera: Shift + Esc + S
и.т.д.

download - предлагает скачать указанный по ссылке файл.
<a download>Ссылка</a>
<a href="Ссылка" download>...</a>

href - задаёт адрес документа, на который необходимо перейти.
<a href="URL">...</a>

hreflang - идентефицирует язык текста по ссылке.
<a href="URL" hreflang="ru">..</a>
Необходим для перемещения на страницу с конкретным языком, когда сайт международный

rel - отношения между ссылаемым и текущим документами.
<a rel="строка">...</a>
answer - ответ на вопрос
chapter - раздел или глава текущего докуменат
и.т.д. http://wiki.whatwg.org/wiki/RelExtensions

tabindex - определяет последовательность перехода между ссылками при нажатии tab.
<a tabindex="число">...</a>

target - имя окна или фрейма, куда браузер будет загружать документ.
<a target="имя окна">...</a>
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self

title - добавляет всплывающую подсказку к тексту ссылки.
<a title="текст">...</a>

type - указывает MIME-тип документа, на который ведёт ссылка.
<a href="URL" type="MIME-тип">...</a>
http://htmlbook.ru/html/value/mime

• <abbr> 
Указывает, что последовательность символов является аббревиатурой.

title - добавляет всплывающую расшифровку аббревиатуры.
<abbr title="Реаьный Хуй Тупо Утопия">РХТУ</abbr>

+ универсальные атрибуты и события


• <address>
Предназначен для хранения информации об авторе и может включать в себя любые элементы HTML.
По умолчанию всегда курсив.

только универсальные атрибуты и события


• <area> - определяет активные области изображения, которые являются ссылками.
Задаёт формы ссылки, её размеры и.т.п. Располагается только внутри тега <map>.

accesskey - переход к области с помощью комбинации клавиш.
<area accesskey="c">
В разных браузерах работает по-разному:
Chrome: Alt + c
Opera: Shift + Esc + S
и.т.д.
 
alt - альтернативный текст для области изображения.
<area href="hui.jpg" alt="хуй">

coords - координаты активной области.
<area coords="X1, Y1,...">
Для различных форм необходимы различный координаты.
Для круга: координаты центра и радиус
Для прямоугольника: координаты одной точки, а затем координаты противоположной
Для многоугольника: последовательно координаты всех вершин

href - задаёт адрес документа, на который необходимо перейти.
<area href="URL">

hreflang - идентефицирует язык текста по ссылке.
<area href="URL" hreflang="ru">
Необходим для перемещения на страницу с конкретным языком, когда сайт международный

shape - определяет форму горячей области
<area shape="circle | default | poly | rect">
default - вся область  

tabindex - определяет последовательность перехода между ссылками при нажатии tab.
<area tabindex="число">

target - имя окна или фрейма, куда браузер будет загружать документ.
<area target="имя окна">
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self

type - указывает MIME-тип документа, на который ведёт ссылка.
<area href="URL" type="MIME-тип">...</a>
http://htmlbook.ru/html/value/mime

+ универсальные атрибуты и события


• <article>
Задаёт содерживое сайта вроде статьи, записи блога, товара в магазине и.т.п.
Обязательно имеет внутри себя заголовок на 1 единицу больше, чем у предыдущего.
По факту является семантическим отростком <div>.


• <aside>
Определяет блок сбоку от контента (сайдбар), семантический отросток <div>.


• <audio>
Отвечает за работу с аудиозаписями, брать только .wav формат. 

autoplay - звук начинает играть сразу после загрузки страницы.
<audio autoplay></audio>

controls - добаляет панель управления к аудиофайлу.
<audio controls></audio>	

loop - повторяет воспроизведение звука после завершения аудио.
<audio loop></audio>

preload - используется для загрузки аудио при загрузке веб-страницы.
<audio preload="none | metadata | auto"></audio>
metadata - только служебная информация (длительность и др.)
auto - всё сразу

src - путь к воспроизводимому файлу, для этой же цели используется <source>.
<audio src="music.wav"></audio>


• <base>
Инструктирует браузер о базовом адресе текущего документа для всех относительных адресов.

href - базовый адрес полного пути.
<base href="http://www.hui.ru/page.html">

target - задаёт имя контекста, которое применяется для всех <a> и <form>.
<base target="имя окна">
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self


• <bdi>
Позволяет фиксировать направление текста на странице.
dir - выбирает направление фиксации тега <p>.
<p dir="rtl"><bdi>שבת</bdi> - суббота</p>

только универсальные атрибуты и события.


• <bdo>
Устанавливает направление вывода текста (преимущественно гле чтение справа налево).

dir - задаёт направление фиксации (rtl | ltr).
<p><bdo dir="rtl">А роза упала на лапу Азора</bdo></p>


• <blockquote>
Предназначен для выделения очень длинных цитат.
По дефолту отображается как выровненный блок с отступами слева и справа, а так же отбивкой сверху и снизу.

cite - указывает на источник цитаты, больше необходим для алгоритмов поиска.
<blockquote cite="vk.com">Телеграм - сила</blockquote>


• <body>
Предназначен для хранения содержимого веб-страницы.

только универсальные атрибуты и события.


• <br>
Устанавливает перевод строки в том месте, где находится тег


• <button>
Создаёт на веб-странице кнопку, в основном используется внутри тега <form>.

accesskey - доступ к элементам формы с помощью комбинации клавиш.
<button accesskey="c">...</button>
В разных браузерах работает по-разному:
Chrome: Alt + c
Opera: Shift + Esc + S
и.т.д.

autofocus - устаналивает, что кнопка получит фокус после загрузки страницы.
<button autofocus>...</button>

disabled - блокирует доступ и изменение элемента
<button disabled>...</button>

form - связывает между собой форму и кнопку.
<form id="hui"></form>
<button form="hui">...</button>

formaction - аналогичен action у <form>, но имеет больший приоритет
<button formaction="hui.js">...</button>

formenctype - устанавливает способ кодирования отправляемых на сервер данных
<button formenctype="application/x-www-form-urlencoded | multipart/form-data | text/plain">...</button>

formmethod - сообщает браузеру метод передачи данных
<button formmethod="get | post">...</button>

formnovalidate - отменяет проверку данных введённых данных (тег required у <input>).
<button formnovalidate>...</button>

formtarget - определяет окно или фрейм, куда будет загружаться результат.
<button formtarget="имя окна">...</button>
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self
имя окна (атрибут name у <iframe>)

name - определяет уникальное имя у кнопки
<button name="hui>...</button>

type - определяет поведение кнопки в <form>
<button type="button | reset | submit">...</button>

value - значение кнопки, отправляющееся на сервер
<button value="hui">...</button>
