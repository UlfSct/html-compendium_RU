• <div>
Базовый блочный элемент для выделения объекта в класс или id.

Только универсальные атрибуты и события


• <span>
В отличие от div, p, table явялется строчным типом, то есть можно задать другой стиль 
для чего-либо, при этом не потеряв уже настроенное расположение элементов.

Только универсальные атрибуты и события.


• <header>
Семантика div для шапки сайта или раздела.


• <nav>
Семантика div для навигации. Запрещается вкладывать в address.


• <main>
Семантика div для главного контента. Не должен включать прочие блоки типа шапки, подвала, боковой панели и.т.п.


• <footer>
Семантика div для подвала сайта или раздела.


• <section>
Семантика div. Отвечает на разделение сайта на секции, содержит заголовок.


• <article>
Семантика div. Задаёт содерживое сайта вроде статьи, записи блога, товара в магазине и.т.п.
Обязательно имеет внутри себя заголовок на 1 единицу больше, чем у предыдущего.


• <aside>
Семантика div. Определяет блок сбоку от контента (сайдбар), семантический отросток <div>.


• <details>.
Семантика div. Используется для хранения информации типа аккордеона.
С помощью тега <summmary> можно задать шапку аккордеона.
<details>
<summary>окси</summary>
гавно, залупа...
</details>

open - по дефолту показывает содержимое блока.
<details open> </details>


• <summary>
Устанавливает заголовок для тега <details>.


• <address>
Предназначен для хранения информации об авторе и может включать в себя любые элементы HTML.
По умолчанию всегда курсив.
