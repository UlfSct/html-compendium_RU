• <form>
Предназначен для обмена данными между пользователем и сервером.
<form action="URL">
   ...
</form>

accept-charset - устанавливает кодировку, в которой сервер может принимать данные.
По умолчанию сохраняет кодировку, установленную на странице.

action - указывает отработчик, к которому обращаются данные формы при отправке их на сервер.

autocomplete - управляет атозаполнением полей форм (даёт команду браузеру).
<form autocomplete="on | off">
   ...
</form>
off используется в основном для паролей, номер карт и каптчи.
По умолчанию стоит on.

enctype - отправляет способ кодирования данных формы при отправке на сервер.
<form enctype="application/x-ww-form-urlencoded | multipart/form-data | text/plain">
   ...
</form>
По умолчанию стоит application/x-www-form-urlencoded

method - сообщает серверу о методе запроса.
<form method="get | post">
   ...
</form>
По умолчанию стоит get. У него есть ограничение в 4 Кб.

name - определяет уникальное имя формы
<form name="penis">
   ...
</form>

novalidate - отменяет дефолтную проверку поля ввода браузером (нужно при отрабатывании ошибок самому).
<form novalidate>
   ...
</form>

target - указывает, где должны открыться страница html после отработки полученных сервером данных.
<form target="имя окна">
   ...
</form>
_blank - в новое окно браузера.
_self - в текущее окно.
_parent - в фрейм-родитель, если такого нет - _self.
_top - отменяет все фреймы и в полном окне, если фреймов нет - _self.
По умолчанию _self.

• <input>
Добавляет элемент формы.

accept="MIME-тип" - устанавливает фильтр на типы файлов в поле загрузки файлов.

accesskey - доступ к элементу формы по горячим клавишам.

alt - альтернативный текст для поля с изображением.

autocomplete - разрешает браузеру автозаполнять форму.

autofocus - устанавливает фокус в поле формы.

checked - при загрузки страницы поле вида radio | checkbox будет уже отмечено.

disabled - отключает доступ к полю.

form - связывает поле ввода с формой по её id.

formaction - аналогичен action у <form>, но имеет больший приоритет. Только для поля submit.
<input formaction="hui.js">

formenctype - устанавливает способ кодирования отправляемых на сервер данных. Только для поля submit.
<input formenctype="application/x-www-form-urlencoded | multipart/form-data | text/plain">

formmethod - сообщает браузеру метод передачи данных. Только для поля submit.
<input formmethod="get | post">

formnovalidate - отменяет проверку данных введённых данных (тег required у <input>). Только для поля submit.
<input formnovalidate>

formtarget - определяет окно или фрейм, куда будет загружаться результат. Только для поля submit.
<input formtarget="имя окна">
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self
имя окна (атрибут name у <iframe>)

list - указывает на список данных datalist.

max - максимальное значение для полей ввода числа или даты.

min - минимальное значение для полей ввода числа или даты.

maxlength - максимальное число символов.

miltiple - несколько файлов загрузки или несколько адресов электронной почты.

name - уникальное имя поля ввода.

pattern - регулирование ввода путём регулярных выражений.

placeholder - подсказка внутри поля ввода, которая исчезает при фокусе.

readonly - аналог disabled, но сохраняются стили.

required - обязательность заполнения поля.

size - ширина текстового поля в количестве моноширинных букв.

src - адрес файла в поле с изображением.

step - шаг изменения числа в поле ввода чисел.

tabindex - последовательность перехода между элементами по Tab.

type - сообщает браузеру тип элемента формы.
<input type="button | checkbox | file | hidden | image | password | radio | reset | submit | text">

value - значение, устанавливаемое в поле ввода при загрузке страницы.


• <textarea>
Элемент формы для написания нескольких строк текста.
В отличие от input, допустимы переносы строк,они сохраняться при передаче данных на отработку.

accesskey - доступ к элементу по горячим клавишам.

autofocus - автоматически устанавливает фокус в текстовое поле.

cols - задаёт ширину поля, значение - количество умещающихся символов в строке.

disabled - отключает текстовое поле.

form - связывает поле с формой по её id.

maxlength - максимальное число символов поля.

name - уникальное имя элемента формы.

placeholder - подсказка внутри поля, которая исчезает при фокусе.

readonly - работает как disabled, но стили при этом сохраняются.

required - обязательность заполнения данного поля формы.

rows - задаёт высоту поля, значение - количество умещающихся символов в столбце.

tabindex - последовательность перехода между элементами формы по Tab.

wrap - работа с переносом текста.
<textarea wrap="soft | hard"></textarea>
soft - текст будет пернесён, но на сервер отправятся только переносы от пользователя.
hard - текст будет перенесён, на сервер отправится со всеми переносами.
По умолчанию - soft.

• <button>
Создаёт на веб-странице кнопку, в основном используется внутри тега <form>.

accesskey - доступ к элементам формы с помощью комбинации клавиш.
<button accesskey="c">...</button>
В разных браузерах работает по-разному:
Chrome: Alt + c
Opera: Shift + Esc + S
и.т.д.

autofocus - устаналивает, что кнопка получит фокус после загрузки страницы.
<button autofocus>...</button>

disabled - блокирует доступ и изменение элемента
<button disabled>...</button>

form - связывает между собой форму и кнопку.
<form id="hui"></form>
<button form="hui">...</button>

formaction - аналогичен action у <form>, но имеет больший приоритет
<button formaction="hui.js">...</button>

formenctype - устанавливает способ кодирования отправляемых на сервер данных
<button formenctype="application/x-www-form-urlencoded | multipart/form-data | text/plain">...</button>

formmethod - сообщает браузеру метод передачи данных
<button formmethod="get | post">...</button>

formnovalidate - отменяет проверку данных введённых данных (тег required у <input>).
<button formnovalidate>...</button>

formtarget - определяет окно или фрейм, куда будет загружаться результат.
<button formtarget="имя окна">...</button>
_blank - в новом окне
_self - в текущем окне
_parent - в фрейм-родитель, если такого нет, то _self
_top - отменяет все фреймы и загружает в полном окне браузера, если фреймов нет - _self
имя окна (атрибут name у <iframe>)

name - определяет уникальное имя у кнопки
<button name="hui>...</button>

type - определяет поведение кнопки в <form>
<button type="button | reset | submit">...</button>

value - значение кнопки, отправляющееся на сервер
<button value="hui">...</button>

• <select>
Позволяет создать выпадающий список. Принимает в себя теги <option>.

accesskey - позволяет перейти к списку по горячим клавишам.

autofocus - список получает фокус сразу после загрузки страницы.
<select autofocus>...</select>

disabled - запрещает взаимодействие со списком.

form - связывает список с формой по её идентефикатору.

multiple - позволяет выбирать несколько значений с помощью Shift и Ctrl.

name - создаёт уникальное имя.

required - задаёт обязательность выбора одной из опций.

size - устанавливает высоту выпадающего списка.

tabindex - определяет последовательность перехода к элементу по Tab.

+ универсальные атрибуты и события.


• <optgroup>
Контейнер для группировки элементов <option>. Не выделяется, но все наследующие сдвигаются вправо.
<select>
   <optgroup>
      <option>...</option>
      ...
   </optgroup>
   <optgroup>
   ...
</select>

disabled - блокирует доступ к группе, обозначая их другим цветом.
<optgroup disabled>...</optgroup>

label - текст, который будет входить в список в виде заголовка группы.

+ универсальные атрибуты и события.


• <option>
Определяет отдельные пункты списка из контейнера <select>.

disabled - блокирует элемент списка для его выбора.

label - предназначен для сокращения длины текста внутри option. 
Пользователю будет показоно содержимое label.

selected - делает текущий элемент списка выделенным.

value - определяет значение для отправки на сервер.

+ универсальные атрибуты и события


• <label>
Устанавливает связь между текстом и элементом формы.
Такая связь необходима для перемещения к полю при нажатии на текст.

accesskey - доступ к элементам формы по горячим клавишам.

for - позволяет ссылаться на id определённого поля ввода.
<input id="penis><label for="penis">хуй</label>


• <fieldset>
Группирует элементы формы (для форм с большим количеством данных).
<form>
   <fieldset>
      <legend>Хуй</legend>
         <input>...
   </fieldset>
   <fieldset>
      <legend>Пизда</legend>
         <input>...
   </fieldset>
   ...
</form>

disabled - блокирует все поля формы в данной группе

title - добавляет всплывающую подсказку при наведении

• <legend>
Отвечает за создание заголовка группы элементов формы <fieldset>
<fieldset>
   <legend>типа группа элементов</legend>
      <input>
      ...
</fieldset>

accesskey - перейти к группе элементов формы по горячим клавишам.

title - добавляет поясняющий всплывающий текст.

• <datalist>
Создаёт список вариантов, которые можно выбирать при наборе в текстовом поле.
Изначально скрыт, виден только при получении полем фокуса или наборе текста.
Список <datalist> связывается через id с помощью атрибута list у <input>
<input list="hui">
<datalist id="hui">
 <option value="hui1">
 <option value="hui2">
</datallist>
